# 📝 AI求职简历助手 — 使用指南

## 这套提示词解决什么问题？

当你让AI帮你写简历时，AI最常见的坏习惯是：**不问你任何细节，直接给你一份看起来很漂亮但全是废话的简历**。结果呢？
- 个人总结全是"性格开朗、工作认真"的模板话术
- 工作经历每条都以"负责"开头，读完不知道你到底做了什么
- 没有任何数字，没有说服力
- 跟你想投的岗位完全不匹配

这套提示词工程强制AI在动手前必须：
1. **分层收集信息** — 先问基础信息、再问经历细节、最后深度挖掘，不会一次性轰炸用户
2. **JD匹配分析** — 对比JD和你的经历，精准找出差距和优化方向
3. **逐模块生成** — 个人信息→个人总结→工作经历→项目经历→技能→教育，每步确认后再继续
4. **STAR法则改写** — 将"负责写接口"变成"主导核心交易链路后端开发，支撑日均100万+订单处理"
5. **ATS优化检查** — 确保简历不会被招聘系统过滤掉
6. **多版本定制** — 针对不同岗位生成不同的简历版本

## 使用流程

### 方式一：从零开始（推荐给第一次做简历的人）

```
1. 打开「千问专用提示词-完整版.txt」
2. 全选复制，粘贴发给千问
3. 告诉千问你的目标岗位（例如："我想找一份高级Java开发工程师的工作"）
4. 千问会开始第一阶段的分层提问
5. 你逐个回答，千问逐步生成简历的各模块
6. 最终得到一份定制化的专业简历
```

### 方式二：优化现有简历

```
1. 打开「千问专用提示词-完整版.txt」
2. 全选复制，粘贴发给千问
3. 附上你现有的简历内容 + 目标JD
4. 千问会先分析匹配度，再逐模块优化
5. 你可以选择"整体优化"或"只优化某个模块"
```

### 方式三：快速出结果

```
1. 打开「千问专用提示词-快速版.txt」
2. 按照提示词里的格式，填入你的信息
3. 把完整的提示词发给千问
4. 一次性拿到优化后的简历
```

## 你需要准备什么？

在开始前，尽量准备好以下信息：

### ✅ 必须有
- 目标岗位名称（越具体越好，不要只说"程序员"）
- 每段工作经历的公司名、职位、起止时间
- 最高学历的学校、专业、学位

### ⭐ 强烈建议有
- 目标岗位的JD（招聘描述）
- 每段工作中你做过的具体事情（不用很专业，口语描述也行，AI会改写）
- 能想起来的数字（提升了多少、负责了多少用户、节省了多少成本）

### 🎁 有的话更好
- 代表性项目的详细描述
- 获奖/证书/开源贡献
- 期望薪资和城市

**如果你还没完全想清楚也没关系** — 完整版提示词会引导你一步步梳理，这是这个Prompt设计的核心价值。

## 核心原则

### 🛑 三不原则
- **不编造**：AI可以美化表达，但不会凭空编造经历和数字
- **不跳步**：严格按阶段推进，每个模块生成后等你确认
- **不替你做决策**：涉及取舍的地方（如写不写某段经历、放不放照片）会列出选项让你选

### ✅ 三必原则
- **必追问**：看到模糊的描述（"做了很多优化"）一定会追问具体做了什么
- **必量化**：每个成果描述都会尝试量化（没有精确数字用"约/近/超"）
- **必确认**：每个模块生成后等你确认，不会一次性输出整份简历让你改

## 常见场景速查

```
# 场景1：应届生，没什么工作经验
→ 使用完整版提示词，在信息收集阶段重点提供：实习经历、课程项目、竞赛经历、GPA

# 场景2：工作3-5年，想跳槽涨薪
→ 使用完整版提示词 + 目标JD，JD匹配分析会告诉你哪些经历最值得突出

# 场景3：跨行业转行
→ 使用完整版提示词，在信息收集阶段说明转行原因，AI会帮你挖掘可迁移技能

# 场景4：只想看看自己简历有什么问题
→ 使用完整版提示词 + 现有简历，跳过信息收集，直接让AI做诊断分析

# 场景5：海投多个相似岗位
→ 先用完整版生成通用版简历，再用第五阶段功能生成多个定制版本
```

| 维度 | 改前 | 改后 |
|------|------|------|
| 个人总结 | 性格描述居多 | 技能+成果+价值主张 |

| HR印象 | 千篇一律 | 6秒内抓住眼球 |
